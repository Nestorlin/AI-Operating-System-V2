# # FILE_NAMING_STANDARD

## Naming Philosophy（命名哲學）

File names must be:

- Predictable
- Searchable
- AI-readable
- Human-readable

---

# Global Rules（全域規則）

## Rule 1
Use uppercase for system files.

系統文件使用大寫。

Example:
SYSTEM_ARCHITECTURE.md

---

## Rule 2
Use underscores instead of spaces.

使用底線取代空格。

Correct:
PROJECT_ROADMAP.md

Wrong:
Project Roadmap.md

---

## Rule 3
Use singular naming when possible.

盡量使用單數命名。

Correct:
PROJECT_SCHEMA.md

Wrong:
PROJECT_SCHEMAS.md

---

## Rule 4
Use ISO date format.

日期格式：

YYYY-MM-DD

Example:
2026-05-12_MEETING.md

---

## Rule 5
Meeting notes format.

格式：

YYYY-MM-DD_CUSTOMER_TOPIC.md

Example:
2026-05-12_FOXCONN_AI_DISCUSSION.md

---

## Rule 6
Avoid non-deterministic names.

禁止：
- final_v2_final_REAL.md
- newfile.md
- temp.md

---

## Rule 7
Archive outdated versions.

舊版必須封存至：

10_ARCHIVE
